# CrewPlay 揪團抓取器

一個幫 CrewPlay 運動媒合平台用的小工具：在 Facebook 揪團社團頁面上，**一鍵把開團貼文抓下來、整理成表格，直接貼進你的 Google 試算表**，不用再一篇一篇手動複製。

---

## 為什麼是這種做法？（先看一下）

Facebook 從 2024 年起，已經**幾乎完全關閉了「用程式在背景自動讀取社團貼文」的官方管道**（Graph API 的社團貼文權限被停用）。市面上號稱能 24 小時全自動抓 FB 社團的服務，幾乎都是：

- 違反 FB 服務條款，
- 容易壞掉（FB 一改版就失效），
- 而且**有讓你帳號被封鎖的風險**。

所以這個工具走的是最穩、最安全的「**半自動**」路線：

> 你（或小編）照常在 FB 瀏覽揪團社團 → 按一下面板按鈕 → 貼文就被整理好、複製到剪貼簿 → 到 Google 試算表貼上。

比純手動快非常多，又不會拿你的帳號去冒險。

---

## 一、安裝（只需做一次，約 5 分鐘）

工具是一個「使用者腳本（userscript）」，要先裝一個瀏覽器外掛來執行它。

### 步驟 1：安裝 Tampermonkey 外掛

到瀏覽器的擴充功能商店搜尋並安裝 **Tampermonkey**（免費）：

- Chrome / Edge：Chrome 線上應用程式商店搜尋「Tampermonkey」
- Firefox：Firefox 附加元件搜尋「Tampermonkey」

### 步驟 2：安裝本工具腳本

1. 點瀏覽器右上角的 Tampermonkey 圖示 → 「**建立新指令碼 / Create a new script**」。
2. 把跳出來的預設內容**全部刪掉**。
3. 打開本資料夾的 [`crewplay-fb-collector.user.js`](./crewplay-fb-collector.user.js)，把**整個檔案內容**複製貼上進去。
4. 按 `Ctrl + S` 儲存。

完成後，之後只要打開 Facebook，右下角就會自動出現一個藍色面板「CrewPlay 揪團抓取器」。

> 第一次更新腳本內容時，重複步驟 2 的 3、4 即可（把舊的內容換成新的再存檔）。

## 二、建立你的 Google 試算表（只需做一次）

本工具的輸出欄位已對齊你現有的「**Crewplay 球館名單**」試算表，順序完全一致，所以抓出來可以直接接著貼。如果是開新表，請把 [`sheet-template.csv`](./sheet-template.csv) 第一列貼到第 1 列當表頭。

欄位順序（**務必與試算表一致**，否則貼上會錯位）：

| 欄位 | 說明 | 是否自動帶入 |
|---|---|---|
| `sport` | 運動類型，羽球 / 桌球 / 排球…（自動辨識關鍵字）| ✅ 自動 |
| `arena_name` | 球館 / 團名（盡量辨識，常需你微調）| 🔶 半自動 |
| `introduce` | 介紹整段：地點 / 時間 / 費用 / 程度 / 用球…（帶入貼文內文，保留換行）| ✅ 自動 |
| `photo` | 照片網址（你自己上傳到 storage 的圖，工具留白）| ⬜ 手動 |
| `assign_url` | FB 貼文連結 | ✅ 自動 |
| `region` | 縣市，高雄市 / 台南市…（自動辨識）| ✅ 自動 |
| `location` | 詳細地址 | ✅ 自動（抓得到才帶）|

`sheet-template.csv` 第 2、3 列有範例資料，看懂後可以刪掉。

> `photo` 欄位工具會留白，因為那是你上傳到自己 storage 的照片網址，FB 貼文裡沒有；抓完後再手動補這一欄即可。

## 三、每天的抓取流程（重點）

1. 打開你的 FB 揪團社團頁面。
2. 右下角面板，依序按：
   - **① 展開全部貼文**：自動點開所有「查看更多」，等 1~2 秒讓內文展開。
   - **② 抓取目前畫面的貼文**：把畫面上的開團貼文抓進工具（面板會顯示「已收集 N 篇」與預覽表）。
   - 想抓更多 → **往下滑動頁面**載入更多貼文 → 再按一次「②」即可繼續累積（會自動去除重複）。
3. 抓得差不多後，按 **③ 複製成表格**。
4. 切到你的 Google 試算表，點一個空白儲存格（例如 A2），按 `Ctrl + V` 貼上 → 整批資料就進去了。

> 備援方式：若複製貼上後 `introduce` 的換行跑掉，可改按「**下載 CSV**」，再到 Google 試算表「檔案 → 匯入 → 上傳」，匯入位置選「附加到目前的工作表」，這樣多行的 `introduce` 會完整保留在同一格。

面板按鈕說明：

- **清空重來**：清掉目前收集的資料，重新開始。
- 標題列可以**拖曳**移動位置，右上角「—」可收合面板。

## 四、常見問題

**Q：抓出來的欄位有些是空的 / 不準？**
A：FB 貼文是純文字、格式很自由，工具用關鍵字去猜（`sport`、`region`、`location`…）。猜不到的欄位會留白，而完整貼文內容都保留在 `introduce` 欄，方便你快速人工補 `arena_name`、`location` 等欄。如果你的社團貼文有固定格式（例如都寫「地點：」「時間：」），辨識率會更高。

**Q：可以自訂要辨識的運動類型或關鍵字嗎？**
A：可以。打開 `crewplay-fb-collector.user.js`，最上面的 `SPORT_KEYWORDS`、`LOCATION_HINTS` 等陣列就是關鍵字清單，照著格式增減即可，存檔後重新整理 FB 頁面就生效。

**Q：面板沒出現？**
A：確認網址是 `facebook.com`、Tampermonkey 是開啟狀態、腳本有存檔；重新整理頁面，等約 1~2 秒。

**Q：之後想做到「自動同步進試算表、再用 Google Cloud 處理」？**
A：那一步需要再加一個 Google Apps Script 的「網路應用程式」當作接收端，工具就能直接把資料 POST 進試算表、省掉貼上的動作。目前先用「複製貼上」版本確認欄位與流程沒問題，要升級隨時跟我說，我再幫你接上。
